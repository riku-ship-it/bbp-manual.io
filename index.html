第一部分：平台概觀與核心價值

BBP 定義與理念

BBP 是企業營運的「數位大腦」，把 組織架構、每日任務、 知識文件 與 管理數據 串在一起，讓管理進度一眼看穿。

  • 工作不瞎忙： 系統幫你記、幫你排，把精力留給真正有產值的事。
  • 經驗不流失： 即使有人請假或離職，做事方法（SOP）都留在平台，新人也能快速接上。

登入準備與環境建議

建議使用最新版 Chrome 或 Edge 瀏覽器，連線穩定的網路環境下操作，體驗最佳。

登入網址： https://my.bbooster.io/login

  • 帳號資訊： 使用顧問為你開通的公司信箱登入。
  • 首次登入： 建議立即於「個人設定」修改預設密碼。
  • 忘記密碼： 可透過登入頁下方「忘記密碼」重新設定。
小提醒：若你長時間未使用，帳號可能會被自動登出，重新登入即可繼續操作。

介面語言設定

BBP 會依瀏覽器環境自動選擇語言。若需手動切換，可依以下步驟調整：

  • 點擊畫面右上角「⚙ 設定」。
  • 找到「Language / 語言」欄位。
  • 從清單中選擇「繁體中文」，畫面會立即切換。

核心運作邏輯：四個模組形成循環

BBP 主要由四個模組組成，形成一個完整的管理循環：

  • 組織圖： 定好誰負責什麼，所有工作從這裡開始。
  • 任務： 把每日工作排進任務與週計畫，確保確實執行。
  • 知識庫： 把做事的方法與規章存起來，新人也看得懂。
  • 儀表板： 把執行結果變成圖表，讓管理者用數據做決策。
組織圖 → 任務 → 知識庫 → 儀表板

第二部分：核心模組詳解

2.1 組織圖 — 企業的骨架

組織圖是整個 BBP 的基礎。與其說是「畫一張圖」，更像是把企業的 職務、責任、成果與指標 正式定義出來。

  • 數據看績效： 職務與儀表板連動，一眼看出部門狀態。
  • 具體成果 (VFP)： 為每個職務定義「最後要交付什麼」。
  • 工作內容與 SOP： 把每天要做的事與步驟寫清楚，新人快上手。

新增組織圖

Step by step

當需要新增一個事業體或重構組織時，可依以下流程建立新版本：

  • 左上角組織圖名稱「下拉選單」 → 點選「＋ 建立組織圖」。
  • 輸入這個組織圖的「名稱」與「說明」，點擊「建立」。
  • 選擇建立模式：
    • 從頭建立：完全自訂，適合全新規劃。
    • 經典版：套用系統預設模板，加快起步。
  • 建立後，可從左上角下拉清單在不同版本間切換。

調整組織圖

組織圖支援即時自動儲存，可用拖曳與點選方式直覺操作。

  • 切換到「編輯模式」後，即可新增、刪除或調整職務格。
  • 點擊職務格右上角「⚙」可進行進階設定（職員、權限、例行任務、指標）。
  • 系統會防呆：只能從最下層開始刪除節點，避免誤刪上層。

2.2 任務 — 執行的動力

任務模組不只是待辦清單，而是把 指派、追蹤、檢視 整合在一起的工作引擎，確保公司目標落實到每天的行動。

  • 透明協作： 支援多位執行者與協作者（CC），資訊不遺漏。
  • 自動統計： 與行事曆連動，系統幫你累計每天工時與任務數。
  • 階層管理： 主管可切換視角，查看部屬任務進度。

任務建立與多模式檢視

建立任務時，建議至少填寫以下欄位：

  • 任務名稱與預期結果： 一句話說清楚要完成什麼。
  • 角色指派： 建立者（1 人）、執行者（可多人）、協作者（可多人）。
  • 時間設定： 開始日期、截止日期（未填開始日的任務不會顯示在行事曆）。

任務可以用三種視角檢視：

  • 任務看板： 以「收件匣 → 已接受 → 進行中 → 已完成」呈現流程。
  • 行事曆： 以時間軸排列，可拖曳任務調整日期或工時長度。
  • 週計畫： 以週為單位，自動整理本週所有任務列表，方便審核。

週計畫提交與主管審核機制

週計畫是讓「主管與同仁目標對齊」的關鍵環節。

  • 職員提交： 進入「週計畫」分頁，確認系統整理出的任務與工時無誤後，按「提交核准」，選擇核准者。
  • 主管審核： 在「待核准」清單中，逐一檢查任務內容與時數。
  • 核准 / 拒絕： 若安排合理則直接核准；若需調整可拒絕並在評論區給具體回饋。

Google 行事曆連動

把會議與行程與 BBP 串起來，可以少開很多分頁。

  • Google 上的會議會自動出現在 BBP 行事曆中。
  • 同步過來的行程會自動計入工時與週計畫。
  • 綁定方式：到「設定 → 個人資料設定 → 整合」，連結 Google 帳號。
一個 BBP 帳號目前僅能綁定一個 Google 帳戶。

2.3 知識庫 — 企業的資產

知識庫就像公司的內部圖書館，所有規章、教學、SOP 都在這裡好好被保存與分送。

  • 資料不難找： 文件集中管理，一搜就有。
  • 新人快上手： 系統會依職務推薦必讀文件。
  • 進度看得見： 可查看同仁是否已完成指定文件的閱讀確認。

建立規章：從分類到發布

建議遵守「先分層，再寫內容」原則：

  • 先在知識庫中建立「資料夾」，例如：規章、職務說明、培訓教材。
  • 進入資料夾後，「新增文件」，幫文件取清楚的名稱並撰寫重點內容。
  • 確認無誤後按「發布」，文件才會正式對其他人開放。

指派學習與閱讀確認

BBP 的特色是可以把文件指派給特定職務或人員：

  • 在文件旁點選「指定學習」，從組織圖中選擇部門或個別員工。
  • 被指派者登入後，可到「待學習」標籤查看需要閱讀的文件。
  • 閱讀完畢後按「學習」，完成該文件的確認。

2.4 儀表板 — 讓數據說真話

儀表板把各種指標集中呈現，讓管理層可以快速了解目前的運營情況。

  • 進度一目了然： 任務完成狀態、專案進度一看圖就懂。
  • 打造專屬畫面： 每位主管可以自己決定要看哪些圖表。
  • 資訊對齊： 儀表板可分享給對應部門，讓大家看同一套數據。

指標建立與設定

指標是儀表板用的「原料」，需要先建立指標，才能放進儀表板中。

  • 建立路徑： 可從組織圖進入特定職務，點選「指標 → 建立」，或在儀表板點「建立 → 指標」。
  • 基本欄位： 指標名稱、負責職務與填寫者、更新頻率（每日 / 每週 / 每月 / 每年）。
  • 圖表類型： 線圖、長條圖、堆疊長條或組合圖，依管理需求選擇即可。

儀表板建立與分享

建立好指標後，就可以組成你自己的儀表板：

  • 在儀表板頁面點選「建立 → 儀表板」，輸入名稱。
  • 使用「＋」加入想要顯示的指標圖表。
  • 建立完成後，可透過右上角選單設定「訪問權限」，依部門或職務分享。

第三部分：管理與技術維護

帳號安全與個人化設定

為了保護公司資料安全，建議定期檢查以下項目：

  • 是否已修改預設密碼，並定期更新。
  • 登入裝置是否為個人電腦或可信任設備。
  • 在個人設定中更新頭像與顯示名稱，方便同事辨識。

常見疑難排解

看不到某些模組或資料？
請先確認你在組織圖中的職務是否正確，被指派的職務權限會決定可見範圍。
畫面反應很慢或卡住？
請檢查網路連線狀態，並嘗試重新整理頁面。若持續發生可聯繫技術支援。

支援管道與多元資源

  • 平台知識庫中有更多操作文件（組織圖、任務、行事曆、指標設定等）。
  • 另有影片教學庫，可透過公司提供的 QR code 或連結觀看。
  • 若遇到系統錯誤或帳號授權問題，請聯繫你的執行顧問或 IT 支援窗口。
